# GB Tile Swapper

**➜ [Try it live](https://pixelpep.com/gbdev/gbtileswaper/)**

A web-based visual tool for creating tile animations and tile-swap effects in **GB Studio** projects. Select tiles visually, generate tilesets, and export ready-to-use GBVM/TileData code — no manual coordinate math required.

---

## Features

### Multi-Set Architecture
- Manage **multiple Sets** of images in a single project using the tab bar at the top
- Each Set has its own LEVEL image, TILESET image, and extra frame images
- **SWAPS** and **SEQUENCES** are shared globally across all sets
- Hovering a swap auto-switches to the set it was created from

### Animation Modes (per Set)
- **FRAME mode**: add separate frame images with the **+FRAME** button — each frame is an independent image file
- **SPRITE SHEET mode**: use a single horizontally-packed sprite sheet and set the frame count via the **FRAMES** counter

### Swap Modes
- **Normal**: create a single swap replacing LEVEL tiles with TILESET tiles
- **BOUNCE (single swap)**: ping-pong animation in one swap — frames play forward then in reverse automatically (1 → 2 → 3 → 2 → 1 → …), perfect for torches, water, breathing effects
- **BOUNCE ×2 (two swaps)**: creates `name_in` (forward) and `name_out` (reverse) as independent swaps, triggerable separately in GBVM — perfect for doors, switches, levers

### Selection Tools
- Click or click-and-drag to select rectangular tile areas
- **SYMSEL**: symmetric selection — mirrors your selection on both images simultaneously
- **KEEPSEL**: keep selection after adding a swap (for building multiple swaps from the same area)

### SWAPS Tab
- Live tile usage stats: `LEVEL X · N swaps Y · total Z / 192 (P%)` — track your GB Studio 192-tile BG limit in real time
- Drag **⋮⋮** handle to reorder swaps; tileset indices update automatically
- Expand **▶** to view and copy the generated code per swap
- Undo support for swap deletion

### Sequence Builder
- Combine multiple swaps into named sequences
- Animated swaps in a sequence are **frame-synchronized** automatically (all swaps change to frame N together)
- Shorter swaps loop automatically when mixed with longer ones
- Drag to reorder items, copy full sequence code with one click

### Preview
- Real-time animated preview of swaps and sequences with FPS control
- Preview window is **vertically resizable** by dragging the bottom edge
- Tileset preview shows the generated tileset (20 tiles per row), also resizable

### Code Export
- **TileData** mode: output for the [TileSwapper plugin](https://github.com/pixelpep) (recommended)
- **GBVM** mode: standard GBVM scripts for use without the plugin
- Type selector: `ts` (tileset) or `bg` (background)

### Project Files (.gbts)
- Save your **entire project** — all sets, all images (embedded as base64), all swaps, all sequences — in a single `.gbts` file
- Reload anytime with **LOAD GBTS**; no external image files needed
- V2 format supports multiple sets; V1 files are automatically upgraded on load

## Quick Start

1. Load **LEVEL.PNG** (your level/background image) and **TILESET.PNG** (your tile graphics)
2. Select matching tiles on both images by clicking or dragging
3. Enter a swap name and click **Add Swap**
4. Repeat for all tile swaps you need
5. Click **PNG** to export the generated tileset, and **CODE** to export the TileData
6. Import the PNG into GB Studio and use the TileData with the TileSwapper plugin
7. Click **SAVE** to save a `.gbts` project file you can reload later

---

## File Format

| File | Description |
|------|-------------|
| `.gbts` | Project file — JSON containing all sets, images (base64), swaps, and sequences |
| `.png` | Generated tileset — import into GB Studio |
| TileData / GBVM | Copy-paste code for use with the TileSwapper plugin or standard GBVM |

---

## TileSwapper Plugin

**GB Tile Swapper** is designed to work with the **TileSwapper** GB Studio plugin by **pixelPep**.
The plugin lets you dynamically replace tiles on the background at runtime using the TileData format generated by this tool.

Without the plugin, switch to **GBVM** output mode to get standard `VM_REPLACE_TILE_XY` scripts.
